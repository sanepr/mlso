# GitHub Copilot Instructions for MLOps Project

## Documentation File Naming Convention

When creating documentation (markdown) files in this project, ALWAYS follow this naming pattern:

### Format
```
docs/YYYYMMDD_HHMMSS_DESCRIPTIVE_NAME.md
```

### Rules
1. **Location**: All docs in `docs/` folder (except root `README.md`)
2. **Timestamp**: Use format `YYYYMMDD_HHMMSS` (e.g., `20251228_143022`)
3. **Name**: UPPERCASE_SNAKE_CASE (e.g., `DOCKER_FIX_SUMMARY`)
4. **Exception**: Never rename `README.md` files

### Examples
```python
# When creating a Docker fix document:
filename = f"docs/{datetime.now().strftime('%Y%m%d_%H%M%S')}_DOCKER_FIX_SUMMARY.md"

# When creating a K8s troubleshooting guide:
filename = f"docs/{datetime.now().strftime('%Y%m%d_%H%M%S')}_K8S_TROUBLESHOOTING.md"
```

### Common Prefixes
- `DOCKER_` - Docker related docs
- `K8S_` or `KUBERNETES_` - Kubernetes docs
- `CI_CD_` - CI/CD pipeline docs
- `TEST_` - Testing docs
- `MODEL_` - ML model docs
- `API_` - API docs
- `MLFLOW_` - MLflow tracking docs

### Before Creating Files
```python
from datetime import datetime

# Get current timestamp
timestamp = datetime.now().strftime("%Y%m%d_%H%M%S")

# Construct proper filename
filename = f"docs/{timestamp}_YOUR_DESCRIPTIVE_NAME.md"
```

## Code Style
- Python: Follow PEP 8, use black formatter (line length 127)
- Test files: Use pytest, prefix with `test_`
- Import sorting: Use isort with black profile
